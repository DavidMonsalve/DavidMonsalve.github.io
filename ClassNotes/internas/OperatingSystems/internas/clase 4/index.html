Clase 1: Introducción a la Materia

¿Qué es un Sistema Operativo?

Es un software que se encarga de gestionar todos los recursos del sistema permitiendo asi la comunicacion entre el usuario y el ordenador.
Su propósito es ofrecer un ambiente en el cual el usuario pueda ejecutar programas.

Objetivos

1) Hacer que el sistema de computo se pueda usar de forma comoda.

2) Usar el hardware de la computadora de forma eficiente.

Funciones básicas del Sistema Operativo

Es "asignador" de recursos de hardware y software. Asigna:


  • Tiempo de CPU.
  • Espacio de memoria.
  • Espacio de almacenamiento para archivos.
  • Dispositivos de E/S.
  • Se encarga de detección y respuesta a errores.
  • Protección.

Es un "administrador" de recursos de hardware y software.


  • Asignandolos a programas y usuarios especificos segun demanda.
  • Resuelve conficlos entre solicitudes de recursos, de manera que el sistema de computo opere de manera justa y eficiente.

Es un "programa de control".


  • Se ocupa especialmente de la operacion y control de dispositivos de E/S.
  • Controla la ejecucion de los programas.

Objetivos (mas detallados) de los Sistemas Operativos

  • Facilidad de Uso

    Forma en que facilita el SO en el uso del sistema, ya sea tanto la parte de hardawre como software.

  • Eficiencia

    Los recursos han de aprovecharse eficientemente. El SO no debe suponer una gran sobrecarga.

  • Capacidad para evolucionar

    Construido de modo que permita una evolución para incorporar nuevas funcionalidades en el futuro.
    Los motivos para la evolucion de los SO son los nuevos tipos de hardware y las actulizaciones del mismo,
    inclusion de nuevos servicios, ya que las necesidades de los clientes van mutando con el pasar del tiempo,
    y resolver errores (o bugs) encontrados en el código.

Singularidad

La singularidad es una caracteristica fundamental del sistema operativo. El sistema operativo no tiene un control externo. Se autocontrola. Esto es lo contrario a lo que pasa con las aplicaciones, estas, son controladas por el SO, quien decide cuando se van a ejecutar, durante cuanto tiempo, etc.

El sistema operativo cuando cuando carga un programa para ejecutar le asigna los recursos necesarios y le entrega el control de todo lo que el maneja. Cuando el programa termina le devuelve el control al SO, pero ese tiempo y esos recursos se los asigna el SO, este le dice cuanto tiempo se va a ejecutar, donde se va a almacenar, cuanto tiempo va a pasar en memoria, etc.
