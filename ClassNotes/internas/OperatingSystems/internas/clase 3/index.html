Clase 3: Procesos e hilos.

Trabajos (jobs)

Conjunto de tareas que realiza una computadora, varía de acuerdo a la generacion a la que nos referimos. En 2da generacion equivale a un lote, en 3era y 4ta a un conjunto de procesos con un objetivo determinado.

Threads, hilos, o procesos ligeros.

Un hilo no es mas que los componentes de un proceso que se pueden ejecutar independientemente.

Concepto de proceso.

Un proceso es un programa en ejecucion. Para ser ejecutado debe estar junto a sus datos en la memoria principal. Para cada proceso, el SO tiene una de estructura que permite identificar los recursos que tiene asignado, llamada BCP. Vendria siendo como una especie de cedula de identidad del proceso.

EL BCP es una estructura que guarda informacion del proceso ¿para qué? para que el sistema operativo lo gestione de manera adecuada. En este bloque de control de proceso vamos a tener informacion de:

  • la identidad del proceso, es decir, quien es el proceso.
  • quienes son los procesos originadores de ese proceso
  • entre otros...

El SO mantiene una tabla de procesos con todos los BCP de todos los procesos.

Componentes de un proceso

Jerarquia de procesos.

La secuencia de creacion de procesos genera un arbol de procesos jerarquico. Existe herencia de padres a hijos.

Entorno del proceso.

Es el conjunto de variables que se le pasan al proceso en el momento de su creacion. Esta formado por una tabla "NOMBRE-VALOR" que se incluye en la pila del proceso.

Grupos de procesos

Los procesos forman grupos que tienen diversas propiedades. Este concepto es de interes puesto que hay determinadas operaciones que se pueden aplicar a todos los procesos de un determinado grupo. Por ejemplo: matar todos los procesos de un mismo grupo.

Informacion del proceso.


La informacion que compone un proceso se organiza en tres grupos:

  • Estado del procesador
  • Imagen de memoria
  • Tablas del sistema operativo

Estado del procesador

Esta formado por el contenido de todos sus registros:

  • Registros generales
  • Resgistros especiales
  • Registros de estado
  • Contador de programa
  • Puntero de pila

Para cada proceso, el estado del procesador recide en: los registros del procesador (en ejecucion) o la memoria (fuera de ejecucion).

Imagen de memoria del proceso.

Es como un mapa de donde esta ubicado un proceso para ser ejecutado.

Esta formada por los espacios de memoria que esta autorizado a utilizar. El proceso solamente puede tener informacion en su imagen de memoria
y no fuera de ella. Está referenciada a memoria virtual o fisica. Solamente tendremos marco
de memoria cuando tengamos gestion de memoria virtual.

Hay cuatro tipos o modelos de imagenes en memoria:

1) Imagen de memoria con un unico segmento de tamaño fijo: Es el modelo mas sencillo.
Su uso se suele restringir a los sistemas sin memoria virtual (MV). El proceso recibe un unico espacio en memoria. el espacio
de memoria no puede variar.

2) Proceso con un unico segmento de tamaño variable: Se emplea raramente. En sistemas sin
memoria virtual los segmentos no pueden crecer a menos que se deje espacio en memoria principal de reserva (desperdicio de memoria).

3) Proceso con un numero fijo de segmentos de tamaño variable: El proceso contiene codigo y datos.
Los datos pueden ser con y sin valor inicial ó dinamicos. Los datos con valor inicial SON ESTATICOS y su valor se fija al cargar el proceso.
Los que no tienen valor inicial tambien son estaticos pero no tiene valor asignado, el SO rellena o no con valores predefinidos esta
zona de datos durante el proceso de carga. Los datos dinamicos se crean y se destruyen segun las indicaciones del
programa.

Usan una estructura de pila residente en memoria por intermedio del puntero de pila o stack-pointer. Es una estructura dinamica durante la ejectucion del proceso.
